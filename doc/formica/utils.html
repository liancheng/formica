5 Miscellaneous issues

5.1 Differences from Racket

In order to use symplifyed syntax for partial application and curring, the arity of some variadic functions, defined in the Racket is changed in Formica.

procedure

(+ x y ...+) → Num

  x : Num
  y : Num
(* x y ...+) → Num
  x : Num
  y : Num
Same as + and * defined in racket/base, but accept at least two arguments.

> (+ 1 2 3)

6

> (+ 1 2)

3

> (+ 1)

#<procedure:curried:+>

> (+)

#<procedure:curried:+>

> (map (+ 1) '(1 2 3))

'(2 3 4)

> (* 1)

#<procedure:curried:*>

> (map (* 2) '(1 2 3))

'(2 4 6)

procedure

(map f lsts ...+) → list?

  f : Fun
  lsts : list?
Same as map, defined in racket/base, but accepts at least one list.

> (map cons '(1 2 3) '(a b c))

'((1 . a) (2 . b) (3 . c))

> (map (* 2) '(1 2 3))

'(2 4 6)

> (map (* 2))

#<procedure:curried:map>

> (map (map (* 2)) '((1 2 3) (2 3 4) (5)))

'((2 4 6) (4 6 8) (10))

procedure

(all? f lsts ...+) → Bool

  f : Fun
  lsts : list?
(any? f lsts ...+) → Bool
  f : Fun
  lsts : list?
Same as andmap and ormap defined in racket/base, but accept at least one list.

> (any? odd? '(1 2 3))

#t

> (all? < '(1 2 3) '(2 3 4))

#t

Some forms could be used as functions

syntax

(or expr ...)

syntax

(and expr ...)

Same as or and and forms defined in racket/base, however, evaluate to binary boolean functions if used not in a head position of application form.

> or

#<procedure:or>

> and

#<procedure:and>

> (or 1 (/ 1 0))

1

> (and #f (/ 1 0))

#f

> (map or '(#t #f #t #f) '(#t #t #f #f))

'(#t #t #t #f)

> (define/c (any p) (foldr (∘ or p) #f))
> (define/c (all p) (foldr (∘ and p) #f))
> (any odd? '(2 4 6 5 8 10))

#t

> (all even? '(2 4 6 5 8 10))

#f

syntax

(==> expr1 expr2)

Represents implication. Equivalent to

(if expr1 expr2 #t).

Returns binary boolean function if used not in a head position of application form.

> ==>

#<procedure:|==>|>

> (==> 'A 'B)

'B

> (==> #f (/ 1 0))

#t

5.2 Operators and functionals

The bindings documented in this section are provided by the formica/tools and formica modules.

procedure

(function? x) → Bool

  x : Any
Returns #t only if x is a function and #f otherwise.

procedure

(id x) → Any

  x : Any
The identity function.

Examples:
> (id 1)

1

> (id 'x)

'x

> (id +)

#<procedure:+>

> ((id +) 1 2)

3

> (id id)

#<procedure:id>

> (id '(a b c))

'(a b c)

procedure

(arg n) → Fun

  n : Ind
Creates a trivial function which returns the n-th argument.

Examples:
> (arg 1)

#<procedure:arg:1>

> ((arg 1) 'x 'y 'z)

'x

> ((arg 2) 'x 'y 'z)

'y

> ((arg 3) 'x 'y 'z)

'z

value

I1 : (arg 1)

I2 : (arg 2)
I3 : (arg 3)

Aliases for grequently used arg calls.

procedure

(const x) → Fun

  x : Any
Creates a trivial function which returns x for any arguments.

Examples:
> (const 'A)

#<procedure:const>

> ((const 'A) 1)

'A

> ((const 'A) 1 2)

'A

> ((const +))

#<procedure:+>

procedure

(composition f ...) → Fun

  f : Fun

value

∘ : composition

Returns a generalized composition of functions f .... Function may have any arity: composition is done as if they all were curried.

In the generalized composition variadic or polyadic functions have minimal possible arity, unless they are greedy.

Function composition has an alias: ∘ (could be entered as \circ + Alt \).

Examples:
> (define-formal (u 1) (b 2) (t 3))
> ((∘ u b) 1 2)

'(u (b 1 2))

> ((∘ b u) 1 2)

'(b (u 1) 2)

> ((∘ t b u) 1 2 3 4)

'(t (b (u 1) 2) 3 4)

> ((∘ t u b) 1 2 3 4)

'(t (u (b 1 2)) 3 4)

> ((∘ u t b) 1 2 3 4)

'(u (t (b 1 2) 3 4))

> ((∘ u b t) 1 2 3 4)

'(u (b (t 1 2 3) 4))

> ((∘ b u t) 1 2 3 4)

'(b (u (t 1 2 3)) 4)

> ((∘ b t u) 1 2 3 4)

'(b (t (u 1) 2 3) 4)

> ((∘ length (filter odd?)) '(1 2 3 4))

2

Composition with nullary function:
> (define-formal (n 0) (u 1) (b 2))
> ((∘ u n))

'(u (n))

> ((∘ b n) 1)

'(b (n) 1)

> ((∘ n u) 1)

nest: Cannot nest function with zero arity!

  function: #<procedure:formal:n>

  position: 2

Composition is associative:
> ((∘ (∘ b t) u) 1 2 3 4)

'(b (t (u 1) 2 3) 4)

> ((∘ b (∘ t u)) 1 2 3 4)

'(b (t (u 1) 2 3) 4)

Composition has left and right neutral element:
> ((∘ b id) 1 2)

'(b 1 2)

> ((∘ id b) 1 2)

'(b 1 2)

In the generalized composition variadic or polyadic functions have minimal possible arity:
> (define (f . x) (cons 'f x))
> (define (g x . y) (list* 'g x y))
> ((∘ f g) 1 2 3 4)

'(f (g 1) 2 3 4)

procedure

(greedy f) → Fun

  f : Fun
For variadic or polyadic function f returns equivalent function having maximal possible arity.

Examples:
> (define-formal f g h)
> ((∘ f g h) 1 2 3 4)

'(f (g (h 1)) 2 3 4)

> ((∘ f (greedy g) h) 1 2 3 4)

'(f (g (h 1) 2 3 4))

> ((∘ remove-duplicates (greedy append)) '(1 2 3) '(2 3 2 4))

'(1 2 3 4)

procedure

(>> f ...) → Fun

  f : Fun
Like ∘ but composes functions f ... in reverse order. Equivalent to

(flipped ∘).

Examples:
> (define-formal f g h)
> ((∘ f g h) 1 2 3 4)

'(f (g (h 1)) 2 3 4)

> ((>> f g h) 1 2 3 4)

'(h (g (f 1 2 3 4)))

procedure

(negated p) → Fun

  p : Fun
(¬ p) → Fun
  p : Fun
Returns the negation of a predicate p.

For function negated there is an alias: ¬ (could be entered as \neg + Alt \).

Examples:
> (negated odd?)

#<procedure:negated:odd?>

> ((negated odd?) 2)

#t

> ((¬ <) 1 2)

#f

procedure

(flipped f) → Fun

  f : Fun
Returns a function which is functionally equivalent to f, but gets it’s arguments in reversed order.

Examples:
> (define snoc (flipped cons))
> snoc

#<procedure:flipped:cons>

> (snoc 1 2)

'(2 . 1)

> ((flipped list) 1 2 3)

'(3 2 1)

procedure

(fif p f g) → Fun

  p : Fun
  f : Fun
  g : Fun
Returns function

x y ... ⟼ (if (p x y ...) (f x y ...) (g x y ...)).

Examples:
> (map (fif odd? (+ 1) id) '(0 1 2 3 4))

'(0 2 2 4 4)

procedure

(andf f g ...) → Fun

  f : Fun
  g : Fun
Returns function

x y ... ⟼ (and (f x y ...) (g x y ...) ...).

Examples:
> (map (andf integer? positive?) '(-3/5 -1 0 2 4.2))

'(#f #f #f #t #f)

procedure

(orf f g ...) → Fun

  f : Fun
  g : Fun
Returns function

x y ... ⟼ (or (f x y ...) (g x y ...) ...).

Examples:
> (map (orf integer? positive?) '(-3/5 -1 2 4.2))

'(#f #t #t #t)

procedure

(fork f g) → Fun

  f : Fun
  g : unary?
(-< f g) → Fun
  f : Fun
  g : unary?
Returns function

x y ... ⟼ (f (g x) (g y) ...).

This function has an alias -<.

Examples:
> ((fork cons sqr) 2 3)

'(4 . 9)

> ((-< + sqr) 1 2 3)

14

procedure

(all-args p) → Fun

  p : Fun
Returns

(fork and p).

Examples:
> ((all-args real?) 2 -3 4.5)

#t

> ((all-args real?) 2 'x 4.5)

#f

procedure

(any-args f) → Fun

  f : Fun
Returns function

(fork or p).

Examples:
> ((any-args real?) '(1 2) 'a "abc" 1-2i)

#f

> ((any-args real?) 'x 2 "abc" 0+8i)

#t

procedure

(curry f arg ...) → (or/c curried? Any)

  f : Fun
  arg : Any
(curried f arg ...) → (or/c curried? Any)
  f : Fun
  arg : Any
Return partially applied (curried) function f, with fixed arguments arg .... Symbols curry and curried are synonyms.

procedure

(curryr f arg ...) → (or/c curried? Any)

  f : Fun
  arg : Any
(r-curried f arg ...) → (or/c curried? Any)
  f : Fun
  arg : Any
Like curry but argumets arg ... are fixed from the right side of argument sequence. Symbols curryr and r-curried are synonyms.

Examples of partial application:
> (curry list 1 2)

#<procedure:curried:list>

> ((curry list 1 2) 3 4)

'(1 2 3 4)

> (map (curried cons 1) '(1 2 3))

'((1 . 1) (1 . 2) (1 . 3))

> (curryr list 1 2)

#<procedure:curried:list>

> ((curryr list 1 2) 3 4)

'(3 4 1 2)

> (map (r-curried cons 1) '(1 2 3))

'((1 . 1) (2 . 1) (3 . 1))

> (curry cons 1 2)

'(1 . 2)

> (curryr cons 1 2)

'(1 . 2)

The curry (curried) and curryr (r-curried) functions correctly reduce the arity of curried function:
> (procedure-arity cons)

2

> (procedure-arity (curried cons))

2

> (procedure-arity (curry cons 1))

1

> (procedure-arity (curryr cons 1))

1

> (procedure-arity (curry + 1 2 3))

(arity-at-least 0)

procedure

(curried? x) → boolean?

  x : Any
Returns #t if x is partially applied or curried function, and #f otherwise.

Examples:
> (curried? (curry cons 1))

#t

> (curried (curry +))

#<procedure:curried:+>

> (curried (curryr +))

#<procedure:curried:+>

> (curried? +)

#f

procedure

(fixed-point f [#:same-test same?]) → Fun

  f : Fun
  same? : (any/c any/c -> boolean?) = equal?
Returns a function

x ⟼ (f (f (f ... (f x))))

which finds a least fixed point of f by iterative application, while result keeps changing in the sense of the same? function.

If function f is not unary, it must return as many values, as it could accept.

Example:
> (define fcos (fixed-point cos))
> (fcos 1)

0.7390851332151607

> (cos (fcos 1))

0.7390851332151607

5.3 Memoization

The bindings documented in this section are provided by the formica/memoize and formica modules.

procedure

(memoized f) → memoized?

  f : Fun
Returns memoized equivalent of the function f.

Examples:
> (define g (memoized current-inexact-milliseconds))
> (g)

1357352871786.622

> (current-inexact-milliseconds)

1357352871787.979

> (g)

1357352871786.622

procedure

(memoized? f) → Bool

  f : Fun
Returns #t if f is a memoized function, and #f otherwise.

Examples:
> (define g (memoized +))
> (memoized? g)

#t

> (memoized? +)

#f

> (g 2 3)

5

syntax

(define/memo (f var ...) body ...)

(define/memo f fun)
Defines memoized function f. If formal variables are not given, the right-hand side of definition fun should evaluate to a function.

Examples:
> (define/memo (f x)
    (display x)
    (* x x))
> (memoized? f)

#t

> (f 2)

2

4

> (f 2)

4

> (f 3)

3

9

syntax

(define/memo/c (f var ...) body)

Defines memoized function f using point-free notation.

5.4 Tagged functions

The bindings documented in this section are provided by the formica/tags module, but not formica.

The formica/tags module provides tools for creating tagged functions.

Tagging allows to distinguish functions and classes of functions.

procedure

(tagged? f) → tagged?

  f : Fun
Returns #t if f is tagged function, and #f otherwise.

procedure

(tag? f) → Sym

  f : tagged?
Returns a tag of a tagged function f.

procedure

(check-tag f t) → Bool

  f : tagged?
  t : Any
Returns #t if f is a tagged function and has tag t, otherwise returns #f.

procedure

(set-tag t [n]) → (Fun -> tagged?)

  t : Sym
  n : (or/c symbol? #f) = #f
Returns an operator, which sets a tag t to given function. If name n is given, tagged function will be renamed.

procedure

(set-tag* t [n]) → (Fun -> Fun)

  t : Sym
  n : (or/c symbol? #f) = #f
Returns an operator, which adds symbol t to function name. The result is not a tagged function. If name n is given, the function will be renamed.

Examples:
> (define t-cons ((set-tag 'tag) cons))
> t-cons

#<procedure:tag:cons>

> (tag t-cons)

'tag

> (check-tag 'tag t-cons)

#t

> (check-tag 'tag cons)

#f

> (tagged? t-cons)

#t

> (define r-cons ((set-tag* 'tag) cons))
> r-cons

#<procedure:tag:cons>

> (check-tag 'tag r-cons)

#f

> (tagged? r-cons)

#f

5.5 Managing the function arity

The bindings documented in this section are provided by the formica/arity, formica/tools and formica modules.

In the refference guide to the Formica language following terminology concerned to function arity is used:

  • Function has fixed arity if it may accept exact finite number of arguments. The arity of function having fixed arity is expressed as positive integer number.

  • Function is called variadic if it may accept different (probably unlimited) number of arguments. The arity of variadic function is expressed with the arity-at-least structure.

  • Function is called polyadic if it may accept different but limited number of arguments. The arity of polyadic function is expressed as a list of positive integers or arity-at-least structure.

procedure

(fixed-arity? f) → boolean?

  f : Any
Returns #t if function f has fixed arity, and #f otherwise.

Examples:
> (fixed-arity? (lambda (x) (+ 2 x)))

#t

> (define (f . x) (cons 'f x))
> (fixed-arity? f)

#f

> (fixed-arity? cons)

#t

> (fixed-arity? +)

#f

procedure

(variadic? f) → boolean?

  f : Any
Returns #t if function f is variadic, and #f otherwise.

Examples:
> (variadic? (case-lambda
               [(x) x]
               [(x . y) (apply + x y)]))

#t

> (define (f . x) (cons 'f x))
> (variadic? f)

#t

> (variadic? cons)

#f

> (variadic? +)

#t

procedure

(polyadic? f) → boolean?

  f : Any
Returns #t if function f is polyadic, and #f otherwise.

Examples:
> (polyadic? (case-lambda

#t

#t

> (polyadic? cons)

> (polyadic? +)

#t

#f

> (nullary? +)

#t

#t

#t

Examples:
> (binary? (case-lambda

procedure

(min-arity f) → (or/c 0 positive?)

  f : Fun
Returns the minimal arity of a function #t.

Examples:
> (min-arity (case-lambda
              [() 0]
              [(x y) (+ x y)]))

0

> (define (f x . y) (list* 'f x y))
> (min-arity f)

1

> (min-arity not)

1

> (min-arity cons)

2

> (min-arity +)

2

procedure

(max-arity f) → (or/c 0 positive? +inf.0)

  f : Fun
Returns the maximal arity of a function #t.

Examples:
> (max-arity (case-lambda
              [() 0]
              [(x y) (+ x y)]))

2

> (define (f x . y) (list* 'f x y))
> (max-arity f)

+inf.0

> (max-arity not)

1

> (max-arity cons)

2

> (max-arity +)

+inf.0

procedure

(inherit-arity f) → (Fun -> Fun)

  f : Fun
Returns the operator which reduces the arity of given function to arity of f, if it is possible.

Examples:
> (procedure-arity
   ((inherit-arity cons) (case-lambda
                           [() 0]
                           [(x . y) (apply + x y)])))

2

> (define (f . x) (cons 'f x))
> (procedure-arity ((inherit-arity +) f))

(arity-at-least 2)
