7 Different utilities

7.1 Tagged functions

 (require formica/tags)

The bindings documented in this section are provided by the formica/tags module, but not formica.

The formica/tags module provides tools for creating tagged functions.

Tagging allows to distinguish functions and classes of functions.

procedure

(tagged? f) → tagged?

  f : Fun
Returns #t if f is tagged function, and #f otherwise.

procedure

(tag? f) → Sym

  f : tagged?
Returns a tag of a tagged function f.

procedure

(check-tag f t) → Bool

  f : tagged?
  t : Any
Returns #t if f is a tagged function and has tag t, otherwise returns #f.

procedure

(set-tag t [n]) → (Fun -> tagged?)

  t : Sym
  n : (or/c symbol? #f) = #f
Returns an operator, which sets a tag t to given function. If name n is given, tagged function will be renamed.

procedure

(set-tag* t [n]) → (Fun -> Fun)

  t : Sym
  n : (or/c symbol? #f) = #f
Returns an operator, which adds symbol t to function name. The result is not a tagged function. If name n is given, the function will be renamed.

Examples:
> (define t-cons ((set-tag 'tag) cons))
> t-cons

#<procedure:tag:cons>

> (tag t-cons)

'tag

> (check-tag 'tag t-cons)

#t

> (check-tag 'tag cons)

#f

> (tagged? t-cons)

#t

> (define r-cons ((set-tag* 'tag) cons))
> r-cons

#<procedure:tag:cons>

> (check-tag 'tag r-cons)

#f

> (tagged? r-cons)

#f

7.2 Managing the function arity

 (require formica/arity)

The bindings documented in this section are provided by the formica/arity, formica/tools and formica modules.

In the refference guide to the Formica language following terminology concerned to function arity is used:

  • Function has fixed arity if it may accept exact finite number of arguments. The arity of function having fixed arity is expressed as positive integer number.

  • Function is called variadic if it may accept different (probably unlimited) number of arguments. The arity of variadic function is expressed with the arity-at-least structure.

  • Function is called polyadic if it may accept different but limited number of arguments. The arity of polyadic function is expressed as a list of positive integers or arity-at-least structure.

procedure

(fixed-arity? f) → boolean?

  f : Any
Returns #t if function f has fixed arity, and #f otherwise.

Examples:
> (fixed-arity? (lambda (x) (+ 2 x)))

#t

> (define (f . x) (cons 'f x))
> (fixed-arity? f)

#f

> (fixed-arity? cons)

#t

> (fixed-arity? +)

#f

procedure

(variadic? f) → boolean?

  f : Any
Returns #t if function f is variadic, and #f otherwise.

Examples:
> (variadic? (case-lambda
               [(x) x]
               [(x . y) (apply + x y)]))

#t

> (define (f . x) (cons 'f x))
> (variadic? f)

#t

> (variadic? cons)

#f

> (variadic? +)

#t

procedure

(polyadic? f) → boolean?

  f : Any
Returns #t if function f is polyadic, and #f otherwise.

Examples:
> (polyadic? (case-lambda
               [(x) x]
               [(x y) (+ x y)]))

#t

> (define (f x (y 2)) '(f x y))
> (polyadic? f)

#t

> (polyadic? cons)

#f

> (polyadic? +)

#f

procedure

(nullary? f) → boolean?

  f : Any
Returns #t if arity of f includes 0, and #f otherwise.

Examples:
> (nullary? (case-lambda
              [() 0]
              [(x) x]
              [(x . y) (apply + x y)]))

#t

> (define (f . x) (cons 'f x))
> (nullary? f)

#t

> (nullary? cons)

#f

> (nullary? +)

#f

procedure

(unary? f) → boolean?

  f : Any
Returns #t if arity of f includes 1, and #f otherwise.

Examples:
> (unary? (case-lambda
              [() 0]
              [(x) x]
              [(x . y) (apply + x y)]))

#t

> (define (f x . y) (list* 'f x y))
> (unary? f)

#t

> (unary? not)

#t

> (unary? cons)

#f

> (unary? +)

#f

procedure

(binary? f) → boolean?

  f : Any
Returns #t if arity of f includes 2, and #f otherwise.

Examples:
> (binary? (case-lambda
              [() 0]
              [(x y) (+ x y)]
              [(x . y) (apply + x y)]))

#t

> (define (f x . y) (list* 'f x y))
> (binary? f)

#t

> (binary? not)

#f

> (binary? cons)

#t

> (binary? +)

#t

procedure

(min-arity f) → (or/c 0 positive?)

  f : Fun
Returns the minimal arity of a function #t.

Examples:
> (min-arity (case-lambda
              [() 0]
              [(x y) (+ x y)]))

0

> (define (f x . y) (list* 'f x y))
> (min-arity f)

1

> (min-arity not)

1

> (min-arity cons)

2

> (min-arity +)

2

procedure

(max-arity f) → (or/c 0 positive? +inf.0)

  f : Fun
Returns the maximal arity of a function #t.

Examples:
> (max-arity (case-lambda
              [() 0]
              [(x y) (+ x y)]))

2

> (define (f x . y) (list* 'f x y))
> (max-arity f)

+inf.0

> (max-arity not)

1

> (max-arity cons)

2

> (max-arity +)

+inf.0

procedure

(inherit-arity f) → (Fun -> Fun)

  f : Fun
Returns the operator which reduces the arity of given function to arity of f, if it is possible.

Examples:
> (procedure-arity
   ((inherit-arity cons) (case-lambda
                           [() 0]
                           [(x . y) (apply + x y)])))

2

> (define (f . x) (cons 'f x))
> (procedure-arity ((inherit-arity +) f))

(arity-at-least 2)
